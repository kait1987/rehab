[재활운동어플리케이션] PRD (최종본)
(동네 기반 재활 헬스장 추천 & 60/90/120분 재활 코스 생성 서비스 — MVP)

1. Overview
1.1 Problem Statement
문제 정의
서울/경기 거주자 중, 스스로 어느 정도 움직일 수 있고 헬스장도 다니지만 “재활운동은 어떻게, 어디서, 무엇을 해야 할지” 모르는 사람들이 많다.

구체적으로:

허리/어깨/무릎/목 등 특정 부위가 불편해 재활운동이 필요하지만,

어떤 운동을 해야 하는지 모르고

동네에서 재활에 적합한 헬스장/운동공간을 찾기 어렵고

병원·센터 기반 재활은 시간·비용 부담이 커 쉽게 접근하지 못한다.

네이버맵/카카오맵 등 기존 지도 서비스로는:

“조용한지, 재활운동에 적합한지, 재활 코치/PT 유무, 평소 혼잡도” 등 재활에 중요한 정보가 구조화되어 있지 않다.

왜 중요한가
재활운동은 꾸준함과 안전성이 핵심인데, “어디서 / 어떻게 시작할지 모름” 때문에 아예 시도하지 못하거나 잘못된 운동으로 통증을 악화시키는 경우가 많다.

서울/경기에는 헬스장·필라테스·소규모 PT샵이 매우 많지만, 재활 친화적 공간 정보가 정리되어 있지 않아 수요-공급 미스매치가 발생한다.

사용자가 당장 오늘 할 수 있는 “동네 기반 + 코스 기반” 솔루션이 있다면 실제 행동 전환(시작/지속)이 크게 증가할 수 있다.

페르소나 (MVP)
Persona A — 직장인 재활러

프로필: 32세, 서울 거주, 사무직(장시간 앉아서 근무), 주 2회 헬스장 이용

상황/배경: 허리(또는 어깨) 통증이 반복되지만 병원 재활은 시간·비용이 부담됨

목표:

통증을 악화시키지 않으면서 운동을 꾸준히 이어가기

집/회사 근처에서 “재활운동하기 좋은 환경”의 헬스장 찾기

페인포인트:

기존 지도/후기에서는 ‘조용함/재활 친화/코치 유무/혼잡도’ 같은 정보가 구조화되어 있지 않음

유튜브/검색은 정보가 너무 많아 “내 상태에 맞는 루틴”을 고르기 어렵고, 잘못하면 통증이 심해질까 불안함

사용 시나리오(핵심):

퇴근 후 60~90분 운동을 계획 → 앱에서 근처 “조용한 재활 친화 헬스장”을 필터로 찾고

채팅형 문답으로 오늘 컨디션에 맞는 루틴을 받아 바로 실행

Persona B — 헬린이~중급, 불안형

프로필: 27세, 서울/경기 거주, 주 1~3회 운동(꾸준히 하려는 편)

상황/배경: 무릎(또는 목) 통증이 가끔 올라오고, 운동을 쉬면 몸이 더 망가질까 걱정

목표:

‘오늘 가능한 시간(60/90/120분)’ 안에서 안전한 루틴을 빠르게 받고 싶음

붐비지 않는 시간대/장소를 알고 싶음(혼잡도 참고)

페인포인트:

PT는 비용이 부담되고, 병원은 “지금 당장 가기엔 애매”

운동을 하긴 해야 하는데 어떤 동작을 해야 안전한지 확신이 없음

사용 시나리오(핵심):

주말에 120분 운동 가능 → 앱에서 코스 생성 후 “이 코스 하기 좋은 근처 헬스장 보기”로 바로 장소 탐색

태그 리뷰로 ‘재활 친화/조용함/장비 상태’를 빠르게 판단

타겟 고객
연령대: 전 연령층(10대 후반~60대), 특히 20–40대 비중 높음

직업: 직장인, 프리랜서, 대학생 등

위치: 서울/경기 (MVP는 강남/홍대 인근 우선)

행동 패턴:

주 2회 이상 헬스장/운동시설 이용, 1–2시간 이상 머무름

시끄럽고 붐비는 곳보다 비교적 조용한 환경 선호

허리/어깨/무릎/목 등 특정 부위 불편함을 경험

병원 재활까지는 부담되지만 “재수술은 피하고 싶다 / 통증을 줄이고 싶다”는 동기 존재

전략적 적합성
1인 개발자로서, 대형 지도/플랫폼이 커버하지 않는 “재활 친화 헬스장 + 초보 재활 루틴” 틈새를 공략 가능하다.

지도/위치 기반 서비스, 챗봇형 문답, 루틴 추천 등의 경험을 포트폴리오/재사용 가능한 자산으로 축적할 수 있다.

향후 재활 코치/트레이너, 재활 특화 센터와의 제휴/수익 모델(광고, 예약 연결 등)로 확장 가능한 기반이 된다.

핵심 전제/가정
사용자는 스스로 기구를 어느 정도 다룰 수 있는 운동 경험자이다(헬린이~중급 수준).

정확한 의학적 진단이 아닌 “불편한 부위/통증 정도/사용 가능한 기구/운동 경험/가능 시간” 수준의 정보만으로도 실용적인 재활 운동 코스 제공이 가능하다.

서울/경기(특히 강남/홍대)에는 재활에 적합하지만 온라인 정보가 부족한 헬스장/공간이 충분히 있다.

네이버맵 API 기반으로 3개월 내 모바일 웹 기반 MVP 런칭이 가능하다.

1.2 Proposed Work
솔루션 개요
동네 기반으로 재활운동하기 좋은 헬스장/운동공간을 추천해주고,

채팅형 문답을 통해 사용자의 몸 상태를 파악해 60/90/120분 재활 운동 코스(기본 90분)를 생성해 주는 서비스.

핵심 기능
위치 기반 주변 재활 헬스장 검색(반경 1km)

현재 위치/선택 위치 기준 반경 1km 내 재활 친화 헬스장/운동공간 리스트 & 지도 표시

실용 정보 필터링

조용한 분위기, 재활 기구 구비, PT/재활 코치 여부, 샤워실, 주차, 가격대 등 필터

(경량) 혼잡도 정보

시간대 기반 + 사용자/점주 수동 업데이트로 “한산/보통/붐빔” 수준의 예상 혼잡도 제공

태그 기반 주민 리뷰

별점 대신 “조용함”, “재활 친화”, “장비 깨끗함”, “복잡함” 등 태그 선택 + 짧은 코멘트

채팅형 재활 코스 추천

불편 부위, 통증 정도, 사용 기구, 운동 경험, 운동 가능 시간에 대한 답변을 기반으로

60/90/120분 재활 운동 루틴(준비–메인–마무리) 생성

1.3 상세 User Flow
1) 첫 실행 & 온보딩
[스플래시/인트로 화면] → [앱 첫 실행] → [온보딩 1 진입]

[온보딩 1 – 서비스 한줄 설명] → [“다음”] → [온보딩 2]

[온보딩 2 – 핵심 기능 요약] → [“시작하기”] → [위치 권한 안내 화면]

[위치 권한 안내 화면] → [“위치 권한 허용하기”] → [OS 위치 권한 팝업]

[OS 팝업 ‘허용’] → [현재 위치 저장] → [홈 화면 진입]

[OS 팝업 ‘거부’] → [홈 화면 진입 + “동네 설정하기” 입력창 노출]

2) 내 주변 재활 헬스장 찾기
[홈 화면] → [“내 주변 재활 헬스장 찾기” 탭] → [지도 & 리스트 화면]

[지도 & 리스트] → [현재 위치 기준 1km 내 장소 조회] → [지도 핀 + 리스트 표시]

[지도 & 리스트] → [필터(조용한 곳/재활기구/주차 등) 선택] → [필터 적용 리스트/핀 재렌더]

[지도/리스트] → [핀 또는 리스트 아이템 탭] → [장소 상세 화면]

[장소 상세] → [정보 스크롤] → [시설, 가격대, 혼잡도, 태그 리뷰 확인]

[장소 상세] → [‘즐겨찾기(♥)’ 탭] → [로컬 또는 계정에 즐겨찾기 저장]

[장소 상세] → [‘리뷰 남기기’ 탭] → [리뷰 작성 화면]

[리뷰 작성] → [태그 ≥1 선택 + (선택) 코멘트 → ‘등록’] → [리뷰 저장 & 상세로 복귀]

3) 내 몸 상태 기반 재활 코스 생성
[홈 화면] → [“내 몸 상태로 재활 코스 만들기” 탭] → [채팅형 코스 생성 화면]

[질문1] “어느 부위가 가장 불편한가요?” → [허리/어깨/무릎/목/기타 중 선택]

[질문2] “통증 정도는 어느 정도인가요? (1~5)” → [1~5 중 탭]

[질문3] “현재 사용 가능한 기구를 선택해 주세요” → [매트/덤벨/머신/밴드/짐볼 등 복수 선택]

[질문4] “평소 운동 빈도는 어느 정도인가요?” → [거의 안 함/주1–2회/주3회 이상 중 선택]

[질문5(선택)] “오늘 운동 시간은 최대 몇 분인가요?” → [60/90/120 중 탭, 미선택 시 기본 90분]

[모든 필수 응답 완료] → [‘코스 생성하기’ 버튼 활성 → 탭] → [코스 생성 로직 실행]

[코스 결과 화면] → [준비–메인–마무리 루틴 표시, 시간/세트/주의사항 확인]

[코스 결과] → [“이 코스 하기 좋은 근처 헬스장 보기” 탭] → [필터 자동세팅된 지도 & 리스트 화면]

[코스 결과] → [‘코스 저장하기’ 탭] → [로컬에 저장 & 홈의 ‘최근 코스’ 영역에 노출]

4) 재방문 플로우
[홈 재진입] → [‘최근 코스’ 카드 탭] → [코스 결과 화면 재진입]

[홈 재진입] → [‘내 주변 재활 헬스장 찾기’ 탭] → [마지막 위치/필터 기준으로 지도 로딩]

1.4 예외 상황(Edge Cases) 요약
위치 권한 거부: 동네 검색/지도 드래그 기반 검색 플로우 제공

네트워크 끊김: 에러 메시지 + 재시도 버튼, 최근 조회 데이터 캐싱 활용

검색 결과 없음: Empty state + 필터 초기화/반경 확장/에디터 추천 노출

외부 API(네이버맵) 응답 없음: 리스트는 그대로, 지도 영역만 에러 메시지 + 새로고침

코스 생성 실패/타임아웃: 안내 + 안전한 기본 코스 Fallback 제공

리뷰 저장 실패: 재시도 버튼, 중복 제출 방지

로컬 저장 불가(프라이빗 모드): 즐겨찾기/최근 코스 저장 불가 안내 후 핵심 기능은 유지

2. Decision / Risk
2.1 Stakeholders
직접 이해관계자:

재활운동이 필요하지만 병원 외 환경에서 스스로 운동하고 싶은 일반 사용자

재활 친화 헬스장/소규모 PT샵/운동공간 운영자

제품 기획·개발·운영을 모두 담당하는 1인 개발자(본인)

간접 이해관계자:

재활 코치, 물리치료사, 트레이너(콘텐츠/루틴 자문)

헬스/재활 관련 온라인 커뮤니티(초기 유입·리뷰 소스)

향후 제휴 파트너: 재활 센터, 병원, 보험사, 기업 복지 담당 등

2.2 Risk & Edge Cases (상세)
Risk 1. 초기 리뷰 부족(cold start)

발생 가능성: 높음 / 영향도: 중간~높음

대응:

운영자 seed 리뷰 30~50개 직접 작성(is_admin_review=true로 구분)

지인 및 헬스/재활 커뮤니티 대상 리뷰 캠페인(소액 리워드)

장소 상세/코스 완료 후 ‘1초 리뷰(태그만)’ 플로우로 작성 장벽 최소화

Risk 2. 장소 정보 수동 관리로 인한 품질 저하

발생 가능성: 높음 / 영향도: 중간

대응:

초기 3개월: 강남/홍대 30~50곳으로 제한

스프레드시트 or 간단 어드민으로 장소 정보 수정 가능하게 구현

“정보 수정 제안하기” 버튼으로 사용자/점주 제보 수집 → 검토 후 반영

월 1회 ‘업데이트일 90일 이상 지난 장소’ 점검 후 수정/비활성화

Risk 3. 재활 코스 안전성·법적 리스크

발생 가능성: 중간 / 영향도: 치명적

대응:

모든 코스 화면·약관에 “의료적 진단·치료가 아닌 일반 운동 정보” 명기

통증 심화 시 즉시 중단·전문의 상담 권고 반복

고강도/고위험 동작 제외, 통증 3/5 이상 시 중단 가이드 포함

재활/트레이닝 종사자 1~2명에게 코스 템플릿 사전 검토 요청

경량 법률 자문(가능 시): 약관/FAQ 문구 점검

Risk 4. 네이버맵 API 한도/설정 이슈

발생 가능성: 낮음 / 영향도: 중간~높음

대응:

강남·홍대 중심, 지도 이동/줌 범위 제한(초기)

API 호출 최소화/캐싱: 같은 위치/반경 중복 호출 방지

호출량 모니터링: 일/주 단위 호출 수 로그 → 한도 70–80% 도달 시 알림

필요 시 유료 플랜 업그레이드 검토

참고: 네이버 지도 API v3는 Client ID 방식이며, NCP에서 Web Dynamic Map 설정이 누락되면 429 에러가 날 수 있어 설정 확인 필요

3. Success Criteria (SMART)
3.1 핵심 지표
등록 재활운동 & 헬스장 수

정의: is_active=true 이고 주소/좌표/운영시간/시설 정보(≥3)가 입력 완료된 장소 수

측정(예): facility_info_count >= 3

목표: 런칭 후 3개월 내 50개 이상

리뷰(태그 리뷰) 수

정의: 삭제되지 않은(is_deleted=false) 리뷰 총 개수(운영자 리뷰 포함)

목표: 3개월 내 총 200개 이상, 이 중 유저 리뷰 100개 이상

월간 활성 사용자(MAU)

정의: 최근 30일 내 핵심 행동(view_map, click_gym_detail, complete_course_generation) 중 1회 이상 수행한 고유 사용자 수

목표: 3개월 차 MAU 300명(1개월 100명, 2개월 200명, 3개월 300명)

3.2 보조 지표(참고)
재활 코스 생성 수: 3개월 내 누적 500회 이상

코스 생성 → 장소 탐색 전환율: 40% 이상

장소 상세 → 리뷰 작성 전환율: 10% 이상

2주 이내 재방문율: 25% 이상

4. Definition of Done (MVP)
4.1 필수 조건(출시 필수)
현재/검색 위치 기준 반경 1km 내 장소 리스트 & 지도 표시

장소 상세: 기본 정보(주소/운영시간/가격대), 시설 정보, 혼잡도, 태그 리뷰 표시

필터: 최소 3개(조용한 곳/재활기구/주차 등) 동작

리뷰 작성: 태그 ≥1 + (선택) 코멘트 저장

채팅형 질문 UI(최소 4~5문항)

60/90/120분 코스 출력(준비–메인–마무리 명시)

“이 코스 하기 좋은 헬스장 보기” → 필터 자동 적용 지도 연동

이용약관 및 개인정보 처리방침 페이지 제공

코스 화면에 “의료행위 아님 / 통증 악화 시 중단·전문의 상담” 문구 명시

기본 에러 로깅/트래킹(FE/BE)

4.2 개선 필요(출시 가능)
로그인 없이 쿠키/로컬스토리지 기반 즐겨찾기·최근 코스 저장

혼잡도: 수동 입력 + 시간대 패턴 기반 단순 “한산/보통/붐빔”

UI/브랜딩 최소 수준

반경 고정(1km), 세밀 반경 조절은 후순위

리뷰 모더레이션은 어드민 수동 삭제 위주(신고 후순위)

4.3 치명적(출시 불가)
지도/리스트에서 장소가 전혀 노출되지 않거나 상세 진입 불가

코스 생성이 자주 실패/에러로 결과를 얻지 못함

약관/안내 없이 진단/치료처럼 오인될 표현 사용

리뷰/코스/즐겨찾기 저장 시 데이터 유실 또는 다른 유저 데이터 혼입

HTTPS 미적용 등 기본 보안 미준수

5. Nope (v1에서 하지 않을 것)
실시간 전문가(의사/물리치료사/트레이너) 1:1 채팅/상담

헬스장/PT 예약 및 결제

센서·CCTV·게이트 연동 ‘실시간’ 혼잡도

iOS/Android 네이티브 앱 동시 개발(모바일 웹/PWA 우선)

AI 포즈 인식/카메라 기반 자세 교정

소셜 기능(팔로우/피드/커뮤니티 게시판)

의료기관 EMR/보험/차트 연동

다이어트/영양/수면 등 라이프스타일 전방위 기능

6. Post-Launch Plan
6.1 v1.1 (Quick Wins – 출시 후 1~2주)
리뷰 UX 개선: 자주 쓰는 태그 상단 고정, 탭 한 번으로 리뷰 완료, 코스 완료 후 리뷰 배너
